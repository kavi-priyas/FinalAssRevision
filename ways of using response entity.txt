## ðŸ”Ž What is `ResponseEntity`?
- A **wrapper** around the HTTP response.

### 2. **Static Factory Methods (`ResponseEntity.ok()`, etc.)**
Spring provides convenient static methods for common cases:

- **OK (200)**:
  ```java
  return ResponseEntity.ok(employee);
  ```
- **Created (201)**:
  ```java

  return ResponseEntity.created(location).body(employee);
  ```
- **No Content (204)**:
  ```java
  return ResponseEntity.noContent().build();
  ```
- **Bad Request (400)**:
  ```java
  return ResponseEntity.badRequest().body("Invalid input");
  ```

### 3. **Empty Response (No Body)**
Sometimes you only want to send a status code:

```java

```

---

### 4. **Custom Status with `status()`**
You can set any status code dynamically:
```java

```
or even:
```java

```

---

### 5. **With Headers**
Add custom headers easily:
```java

```

## âš¡ Best Practice
- Use **static methods** (`ok()`, `created()`, `noContent()`) for readability.
- Use **constructor** only when you need full manual control (headers + body + status).
- Always return meaningful **status codes** (201 for create, 404 for not found, 204 for delete).
